# 핵심 개념

ZEC Toolkit을 이해하기 위한 4가지 핵심 개념을 안내합니다.

## 학습 순서

| 순서 | 주제 | 핵심 질문 |
|------|------|----------|
| 1 | [ZEC 오케스트레이터란?](/core-concepts/what-is-zec) | 에이전트 위임은 어떻게 동작하는가? |
| 2 | [SPEC 기반 개발](/core-concepts/spec-based-dev) | 요구사항을 어떻게 체계적으로 정의하는가? |
| 3 | [개발 방법론 (DDD/TDD)](/core-concepts/ddd-tdd) | 기존 코드를 안전하게 개선하는 방법은? |
| 4 | [TRUST 5 품질 게이트](/core-concepts/trust-5) | 코드 품질을 어떻게 보장하는가? |

SPEC으로 **무엇을 만들지** 정하고, DDD/TDD로 **안전하게 만들고**, TRUST 5로 **품질을 검증**하는 흐름입니다.

각 문서는 독립적으로 읽을 수 있지만, 순서대로 읽으면 전체 철학이 연결됩니다.
