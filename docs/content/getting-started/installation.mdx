# 설치

ZEC Toolkit을 Claude Code에 설치하는 방법을 안내합니다.

import { Callout, Steps, Tabs } from 'nextra/components'

## 사전 요구사항

| 항목 | 요구사항 | 확인 방법 |
|------|----------|-----------|
| **Claude Code** | 최신 버전 | `claude --version` |
| **Git** | 2.x 이상 | `git --version` |
| **Node.js** | 18 이상 (선택) | `node --version` |

<Callout type="warning">
Claude Code가 설치되어 있지 않다면 먼저 [Claude Code 공식 문서](https://docs.anthropic.com/en/docs/claude-code)를 참고하여 설치하세요.
</Callout>

### 마켓플레이스 등록

Claude Code에서 `/plugin` → **Marketplaces** 탭 → **Add** 선택 후 입력:

```
lemonbalms/zec-toolkit
```

### 플러그인 설치

마켓플레이스 등록 후 **Discover** 탭에서 `zec-toolkit`을 찾아 **Install** 합니다.

설치 범위를 선택할 수 있습니다:

| 범위 | 설명 | 저장 위치 |
|------|------|----------|
| user | 모든 프로젝트에서 사용 (기본) | `~/.claude/` |
| project | 팀 공유 (git에 포함) | `.claude/settings.json` |
| local | 현재 저장소만 (gitignored) | `.claude/settings.local.json` |

### 설치 확인

Claude Code에서 `/zec` 명령어가 인식되는지 확인합니다:

```
> /zec
```

ZEC 오케스트레이터가 응답하면 설치가 완료된 것입니다.

</Steps>

## 포함되는 항목

설치 시 다음이 자동으로 Claude Code에 등록됩니다:

| 항목 | 수량 | 설명 |
|------|------|------|
| **Skills** | 10개 | ZEC 오케스트레이터 + Harness 스킬 9종 |
| **Rules** | 6개 | 핵심 원칙, 코딩 표준, 워크플로우 규칙 |
| **Templates** | 15개 | `.zec/config/` YAML 설정 파일 |
| **Agents** | 3개 | executor, deep-executor, explore |
| **Bridge** | 8개 | Node.js 실행 엔진 (MCP 서버, 키워드 감지 등) |

<Callout type="info">
Rules는 플러그인 설치 시 자동으로 활성화됩니다. Skills는 해당 작업 컨텍스트에서 자동 트리거되거나 `/zec` 명령으로 수동 실행할 수 있습니다.
</Callout>

## 관리

| 작업 | 방법 |
|------|------|
| 설치/제거/업데이트 | `/plugin` → **Installed** 탭 |
| 마켓플레이스 추가/제거 | `/plugin` → **Marketplaces** 탭 |
| 로딩 오류 확인 | `/plugin` → **Errors** 탭 |

<Callout type="warning">
제거 시 프로젝트의 `.zec/` 디렉토리는 삭제되지 않습니다. 설정을 완전히 제거하려면 수동으로 삭제하세요.
</Callout>

## 문제 해결

### 마켓플레이스 등록 실패

```
Marketplace file not found at ...marketplace.json
```

GitHub URL 형식을 확인합니다. `lemonbalms/zec-toolkit` (owner/repo) 형식을 사용하세요.

### 명령어를 찾을 수 없음

```
> /zec
Error: Unknown command
```

`/plugin` → **Installed** 탭에서 `zec-toolkit`이 있는지 확인합니다. 없다면 재설치합니다.

### 스킬이 로드되지 않음

`/plugin` → **Errors** 탭에서 오류를 확인합니다. `plugin.json`에 스킬 경로가 올바른지 점검합니다:

```json
{
  "skills": "./skills/"
}
```
