# 초기 설정

프로젝트에 ZEC Toolkit 설정을 적용하는 방법을 안내합니다.

import { Callout, Steps, FileTree } from 'nextra/components'

## 개요

ZEC Toolkit은 프로젝트 루트의 `.zec/config/sections/` 디렉토리에 YAML 파일로 설정을 관리합니다.
설치 후 프로젝트별로 이 설정을 초기화해야 합니다.

## 초기화 방법

<Steps>

### 설정 디렉토리 생성

```bash
mkdir -p .zec/config/sections .zec/specs .zec/reports
```

### 필수 설정 파일 작성

최소한 4개의 YAML 파일을 설정해야 합니다:

#### user.yaml — 사용자 정보

```yaml filename=".zec/config/sections/user.yaml"
user:
  github_username: your-username
  name: your-project-name
```

#### language.yaml — 언어 설정

```yaml filename=".zec/config/sections/language.yaml"
language:
  conversation_language: ko
  conversation_language_name: ko
  agent_prompt_language: en
  git_commit_messages: ko
  code_comments: ko
  documentation: ko
  error_messages: en
```

#### quality.yaml — 품질 설정

```yaml filename=".zec/config/sections/quality.yaml"
quality:
  development_mode: ddd    # ddd 또는 tdd
  test_coverage_target: 85
  enforce_quality: true
```

#### execution.yaml — 실행 설정

```yaml filename=".zec/config/sections/execution.yaml"
execution:
  default_mode: ultrawork
  model_routing:
    simple: haiku
    standard: sonnet
    complex: opus
  persistence:
    max_iterations: 50
    state_dir: .zec/state
```

### CLAUDE.md 연동

프로젝트의 `CLAUDE.md`에 ZEC 실행 지침을 추가합니다. 템플릿은 [설정 가이드](/configuration/yaml-config)에서 확인할 수 있습니다.

### 설정 확인

Claude Code에서 설정이 올바르게 로드되는지 확인합니다:

```
> /zec
```

</Steps>

## 필수 vs 선택 설정

| 파일 | 필수 | 설명 |
|------|:----:|------|
| `user.yaml` | ✅ | 사용자/프로젝트 식별 |
| `language.yaml` | ✅ | 대화/코드/문서 언어 |
| `quality.yaml` | ✅ | 품질 게이트 설정 |
| `execution.yaml` | ✅ | 실행 모드 + 모델 라우팅 |
| `project.yaml` | | 프로젝트 메타데이터 |
| `system.yaml` | | 시스템 버전 정보 |
| `workflow.yaml` | | 워크플로우 세부 설정 |
| `context.yaml` | | 컨텍스트 관리 설정 |
| `git-convention.yaml` | | 커밋 메시지 규칙 |
| `git-strategy.yaml` | | 브랜치 전략 |
| `llm.yaml` | | LLM 모델 설정 |
| `mx.yaml` | | MX 태그 설정 |
| `ralph.yaml` | | Ralph 루프 설정 |
| `state.yaml` | | 상태 관리 설정 |
| `statusline.yaml` | | 상태 표시줄 설정 |

<Callout type="info">
선택 설정은 필요할 때 추가하면 됩니다. 템플릿은 `templates/zec-config/sections/`에 있습니다.
</Callout>

## .gitignore 설정

`.zec/` 디렉토리에서 상태 파일은 무시하고 설정은 커밋하는 것을 권장합니다:

```gitignore filename=".gitignore"
# ZEC 상태 파일 (세션별, 무시)
.zec/state/
.zec/reports/

# ZEC 설정 (커밋)
# .zec/config/ → 커밋 대상
# .zec/specs/ → 커밋 대상
```
