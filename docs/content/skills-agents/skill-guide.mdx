# 스킬 가이드

스킬은 ZEC Toolkit의 **재사용 가능한 지식 모듈**입니다. 특정 기술 스택이나 워크플로우에 대한 전문 지식을 캡슐화합니다.

import { Callout, FileTree } from 'nextra/components'

## 스킬 시스템 개요

```
사용자 요청
    │
    ▼
ZEC 오케스트레이터
    │
    ├─ 자동 트리거 ─────▶ 키워드 매칭으로 스킬 활성화
    │                     예: FastAPI 코드 → harness-fastapi
    │
    └─ 수동 실행 ────────▶ /zec plan, /zec run 등
                           예: /zec → ZEC 오케스트레이터 스킬
```

## SKILL.md 구조

모든 스킬의 진입점 파일입니다:

```yaml
---
name: harness-fastapi
description: FastAPI 백엔드 개발 가이드라인
allowed-tools: Read, Write, Edit, Bash, Grep, Glob
argument-hint: "[task description]"
---
```

| 필드 | 설명 |
|------|------|
| `name` | 스킬 고유 식별자 |
| `description` | 트리거 매칭에 사용되는 설명 |
| `allowed-tools` | 사용 가능한 도구 목록 |
| `argument-hint` | 사용자 입력 힌트 |

## 트리거 방식

### 자동 트리거

스킬의 `description`과 `triggers.keywords`에 매칭되는 키워드가 감지되면 자동 활성화됩니다.

예시: FastAPI 코드를 작성할 때 `harness-fastapi` 스킬이 자동으로 로드되어 DDD 패턴, 레이어드 아키텍처 가이드라인을 적용합니다.

### 수동 실행

`/zec` 명령어로 직접 실행합니다:

```bash
/zec plan "인증 API"     # ZEC 오케스트레이터 → plan 워크플로우
```

## 스킬 종류

| 종류 | 예시 | 역할 |
|------|------|------|
| **오케스트레이터** | `zec/` | 워크플로우 라우팅 + 에이전트 위임 |
| **Harness** | `harness-fastapi/` | 기술 스택 가이드라인 + 패턴 |
| **Tool** | `zec-tool-ast-grep/` | 특정 도구 사용법 |
